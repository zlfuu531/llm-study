# 答案版 12：大模型训练系统与分布式训练

对应 `03_高频题单100题.md` 的 261-280 题。

## 261. 大模型训练显存由哪些部分组成？

主要由参数、梯度、optimizer state、activation、临时 buffer、通信 buffer 和碎片组成。混合精度 Adam 全参训练常用粗估是参数 2P、梯度 2P、FP32 master weights 4P、Adam m/v 8P，合计约 16P bytes，还没算 activation。

## 262. 7B/14B 全参 Adam 训练怎么粗估显存？

按 16 bytes/param 粗估参数、梯度和优化器状态：7B 约 112GB，14B 约 224GB。实际还要加 activation、通信 buffer、临时 workspace 和碎片，所以不能只按权重 FP16 的 14GB/28GB 算训练显存。

## 263. DDP 的核心流程是什么？

每张卡保存完整模型，处理不同 mini-batch，反向得到梯度后做 all-reduce，使每张卡拿到同步后的平均梯度，再各自更新同样的参数。DDP 简单稳定，适合模型单卡放得下、想扩 batch 和吞吐的场景。

## 264. DDP 为什么不能解决单卡放不下模型的问题？

因为 DDP 每张卡都有完整参数、梯度和 optimizer state，只切数据 batch，不切模型状态。卡数增加能提升数据吞吐，但每卡显存并不会因为 DDP 自动下降，所以单卡放不下时要考虑 ZeRO/FSDP、TP、PP 或 LoRA。

## 265. ZeRO-1、ZeRO-2、ZeRO-3 分别切什么？

ZeRO-1 切 optimizer state，ZeRO-2 在 ZeRO-1 基础上再切 gradients，ZeRO-3 连 parameters 也切。阶段越高每卡显存越省，但通信和实现复杂度也更高，尤其 ZeRO-3 需要参数 all-gather 和梯度 reduce-scatter。

## 266. FSDP 和 ZeRO-3 怎么理解？

FSDP 可以理解成 PyTorch 生态里的 fully sharded data parallel，思路和 ZeRO-3 接近：切分参数、梯度和优化器状态。计算某层前 all-gather 参数，反向后 reduce-scatter 梯度，用通信换显存。关键要调 wrap 粒度、prefetch、reshard 和 checkpoint。

## 267. Tensor Parallel 切的是什么？

Tensor Parallel 切的是层内矩阵计算，比如 attention 的 QKV projection、output projection、MLP 的 up/gate/down projection。它适合单层 hidden 或 intermediate 很大、单卡放不下或 matmul 压力太高的情况，但每层都有通信，最好放在节点内高速互联上。

## 268. Pipeline Parallel 切的是什么？

Pipeline Parallel 切的是模型层，把不同层放到不同 GPU。为了减少空闲，会把 batch 切成 micro-batches 做流水执行。核心问题是 pipeline bubble、stage imbalance 和 activation 跨 stage 传输。

## 269. 3D 并行怎么组合？

3D 并行通常是 DP × TP × PP。TP 切单层矩阵，PP 切层，DP 复制多个 TP×PP 模型副本处理不同数据。例如 64 卡可以设 TP=4、PP=4、DP=4。实际选择要考虑模型大小、单层大小、层数、global batch 和节点内/跨节点网络。

## 270. Activation checkpointing 为什么省显存？

它前向时不保存所有中间 activation，只保存少量 checkpoint；反向时重新计算缺失的中间结果。它省的是 activation 显存，不省参数和 optimizer state，代价是训练变慢。长序列、大 batch、深模型时很有用。

## 271. 梯度累积和 global batch 的关系是什么？

`global_batch = micro_batch_per_gpu * data_parallel_size * grad_accum_steps`。梯度累积通过多次 micro-batch 前向/反向后再 optimizer step，模拟更大 batch，降低单次显存压力。注意 loss 缩放、学习率、warmup 和累积步数要匹配。

## 272. FP16、BF16、FP8 训练有什么区别？

FP16 省显存且快，但动态范围小，容易梯度下溢，常需要 loss scaling。BF16 动态范围接近 FP32，训练更稳，很多 LLM 训练优先用 BF16。FP8 更省带宽和存储，但依赖硬件和框架，需要 scaling 与数值稳定策略，不能简单当成无损替代。

## 273. 训练通信瓶颈常出现在哪里？

DDP 主要是梯度 all-reduce；ZeRO/FSDP 是参数 all-gather 和梯度 reduce-scatter；TP 是层内通信；PP 是 pipeline bubble 和 stage 间传输；MoE expert parallel 是 token dispatch 的 all-to-all。小 batch、跨节点带宽不足和 overlap 不好都会放大通信瓶颈。

## 274. MoE 训练为什么难？

MoE 每个 token 只激活部分 experts，参数容量大但计算相对稀疏。难点是路由塌缩、负载不均、capacity 溢出、all-to-all 通信重、专家利用率不均和训练不稳定。需要 load balancing loss、合理 capacity factor 和 expert parallel 调度。

## 275. Expert Parallel 通信是什么？

Expert Parallel 把不同 experts 放到不同 GPU。每层 MoE 时，要按 router 结果把 token 发送到对应 expert 所在 GPU，算完再发回原位置，核心通信通常是 all-to-all。瓶颈是 token 分布不均和跨节点 all-to-all 成本。

## 276. 训练 OOM 怎么排查？

先判断显存花在哪里：参数/optimizer state 太大用 ZeRO/FSDP、LoRA 或 offload；activation 太大用 checkpointing、减 batch/seq、FlashAttention；临时 buffer 太大就调 kernel 或分块；碎片问题要看 allocator、batch shape 和重启，不要只靠 empty_cache。

## 277. 训练吞吐低怎么排查？

看 step time 分解、GPU 利用率、dataloader、通信时间、kernel 时间、micro-batch 数、PP bubble 和网络带宽。根据并行策略定位：DDP 看 all-reduce，FSDP/ZeRO 看 all-gather/reduce-scatter，TP 看层内通信，MoE 看 all-to-all。

## 278. Loss NaN 常见原因有哪些？

常见原因包括学习率过大、FP16 溢出/下溢、loss scaling 异常、梯度裁剪缺失、坏数据、mask 或 label shift 错、梯度累积缩放错、分布式同步异常。排查时先缩小复现，用更小 batch、BF16/FP32、检查数据和 loss，再看梯度范数。

## 279. 全参微调、LoRA、FSDP/ZeRO 怎么选？

如果只是行为适配且显存有限，优先 LoRA/QLoRA；如果需要深度改变模型能力且数据和算力足够，考虑全参微调。全参时如果单卡状态放不下，用 FSDP/ZeRO 切参数、梯度和优化器状态。选择要看效果需求、显存、训练成本、上线风险和评估结果。

## 280. 训练系统项目 8 分钟深挖版讲什么？

按“模型规模 -> 数据规模 -> 显存估算 -> 并行策略 -> batch/学习率 -> 稳定性 -> 吞吐 -> checkpoint -> 评估”讲。比如说明 14B 全参训练按 16 bytes/param 粗估需要 224GB 以上状态显存，所以用了 FSDP/ZeRO、activation checkpointing 和 BF16；再讲 step time、tokens/s、OOM 排查、loss 曲线、验证集指标和 bad case。
