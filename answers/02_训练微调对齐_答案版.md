# 答案版 02：训练、微调与对齐

这一份对应 `03_高频题单100题.md` 的第 21-40 题。

## 21. 预训练和 SFT 的区别是什么？

**面试答案：**  
预训练让模型从大规模语料中学习语言规律和世界知识，目标通常是 next token prediction。SFT 是用高质量指令-回答数据继续训练，让模型学会按人类指令回答。预训练解决“会不会”，SFT 解决“会不会按要求回答”。

**追问：SFT 会不会让模型学新知识？**  
可以学到一部分，但 SFT 更主要改变行为模式和输出格式。大量事实知识更新更适合 RAG 或继续预训练。

## 22. Causal LM 和 Masked LM 的区别是什么？

**面试答案：**  
Causal LM 根据左侧上下文预测下一个 token，不能看未来，适合生成任务。Masked LM 随机 mask 输入中的部分 token，让模型根据双向上下文预测被 mask 的 token，适合理解任务。

**代表模型：**

- Causal LM：GPT、LLaMA、Qwen。
- Masked LM：BERT、RoBERTa。

## 23. 为什么 SFT 通常只对 answer 部分算 loss？

**面试答案：**  
因为我们希望模型学习如何回答，而不是学习复述 prompt。如果对 instruction 和 input 也算 loss，模型会被训练去预测用户问题本身，浪费学习信号，还可能影响指令跟随效果。

**工程实现：**  
构造 labels 时，把 prompt 部分设为 `-100`，让 cross entropy 忽略，只对 answer token 计算 loss。

## 24. 数据清洗对大模型训练为什么重要？

**面试答案：**  
大模型对数据质量很敏感。低质量、重复、冲突、带毒或格式混乱的数据会导致模型幻觉、重复、风格变差、安全风险增加。尤其 SFT 和偏好对齐阶段，少量脏数据就可能显著影响模型行为。

**清洗重点：**

- 去重。
- 去低质。
- 脱敏。
- 格式统一。
- 去污染测试集的数据。
- 过滤违法、安全和隐私风险内容。

## 25. LoRA 的原理是什么？

**面试答案：**  
LoRA 冻结原模型权重，只训练一个低秩更新分支。对原线性层 `Wx`，LoRA 增加 `B(Ax)`，其中 A 和 B 是低秩矩阵，参数量远小于完整 W。

```text
y = Wx + scale * B(Ax)
```

**直觉：**  
预训练模型已经有通用能力，下游任务只需要在少数方向上调整，因此权重更新可以用低秩矩阵近似。

## 26. LoRA 的 rank 和 alpha 分别控制什么？

**面试答案：**  
rank 控制低秩更新的容量，rank 越大，可训练参数越多，表达能力越强，但更容易过拟合、显存也更高。alpha 控制 LoRA 分支的缩放强度，常用 `alpha / rank` 作为 scale。

**实际选择：**  
rank 不是越大越好，要看数据量、任务复杂度和验证集效果。

## 27. LoRA 通常加在哪些层？

**面试答案：**  
常见加在 attention 的 `q_proj`、`v_proj`，也可以加 `k_proj`、`o_proj`，以及 FFN 的 `up_proj`、`down_proj`、`gate_proj`。只加 Q/V 参数少，成本低；加更多模块效果可能更好，但训练和存储成本也更高。

**项目回答：**  
如果是资源有限的业务微调，我会先从 q/v LoRA 开始，验证效果后再考虑 attention 全投影或 FFN。

## 28. LoRA、Adapter、Prefix Tuning 有什么区别？

**面试答案：**

- LoRA：在权重更新上加低秩分支，推理时可 merge。
- Adapter：在网络层之间插入小模块，会增加推理路径。
- Prefix Tuning：给每层 attention 加可训练 prefix key/value，不直接改权重。

**取舍：**  
LoRA 实践最常见，容易合并部署；Adapter 模块化强但推理有额外层；Prefix 对生成任务有用，但可解释和调参更复杂。

## 29. QLoRA 为什么省显存？

**面试答案：**  
QLoRA 把冻结的基座模型用 4-bit 等低比特量化存储，只训练 LoRA 参数。基座权重不需要全精度训练状态，显存主要省在模型权重和 optimizer state 上。

**误区：**  
不是所有计算都变成 4-bit，也不是 LoRA 参数一定 4-bit。通常 LoRA 参数仍用较高精度训练。

## 30. 全量微调和参数高效微调怎么选？

**面试答案：**  
数据量大、任务和基座差异大、资源充足时可以考虑全量微调；资源有限、任务是领域适配、希望多任务保存多个小权重时优先 LoRA/QLoRA。大多数业务学习或实践项目，用 LoRA/QLoRA 更现实。

**补充：**  
如果只是补事实知识，优先 RAG；如果是改变回答风格、格式和任务行为，再考虑微调。

## 31. RLHF 的完整流程是什么？

**面试答案：**

1. 先用 SFT 训练一个指令模型。
2. 对同一 prompt 采样多个回答，让人类标注偏好。
3. 用偏好数据训练 Reward Model。
4. 用 PPO 等强化学习方法优化策略模型，使它获得更高 reward。
5. 用 KL 约束防止模型偏离原模型太远。

**一句话：**  
RLHF 是用人类偏好训练奖励模型，再用强化学习让语言模型更符合偏好。

## 32. Reward Model 怎么训练？

**面试答案：**  
Reward Model 输入 prompt 和 response，输出一个标量分数。训练数据通常是 chosen/rejected 偏好对，目标是让 chosen 的 reward 高于 rejected。

常见损失形式：

```text
loss = -log sigmoid(r_chosen - r_rejected)
```

**注意：**  
Reward Model 学的是偏好比较，不是直接生成答案。

## 33. PPO 在 RLHF 里起什么作用？

**面试答案：**  
PPO 用 Reward Model 给出的奖励来更新策略模型，让模型生成更高 reward 的回答。同时 PPO 会限制策略更新幅度，避免模型一次更新偏离太大。

**为什么复杂：**  
它涉及采样、reward、advantage、KL、value/critic 等，比普通监督学习更难调。

## 34. 为什么 RLHF 需要 KL 约束？

**面试答案：**  
如果只最大化 Reward Model，模型可能钻奖励模型漏洞，生成高 reward 但质量差或不自然的回答。KL 约束让新模型不要偏离 SFT/reference model 太远，保持语言质量和稳定性。

**一句话：**  
KL 是防止 reward hacking 和模型漂移的安全绳。

## 35. DPO 的核心思想是什么？

**面试答案：**  
DPO 直接用 chosen/rejected 偏好对优化模型，不显式训练 Reward Model，也不需要 PPO 那样在线强化学习。它让模型相对 reference model 更偏向 chosen，而不是 rejected。

核心形式：

```text
loss = -log sigmoid(beta * (pi_logratio - ref_logratio))
```

其中 `pi_logratio` 是当前模型对 chosen/rejected 的 log probability 差，`ref_logratio` 是参考模型的差。

## 36. DPO 和 PPO 的区别是什么？

**面试答案：**  
PPO 是强化学习方法，需要 Reward Model、采样、advantage 和 KL 控制，训练复杂。DPO 是直接偏好优化，把偏好数据转成类似监督学习的目标，不显式训练 Reward Model，流程更简单、稳定性更好。

**但要注意：**  
DPO 不是永远比 PPO 好。效果取决于偏好数据质量、任务类型和训练配置。

## 37. DPO 为什么需要 reference model？

**面试答案：**  
reference model 是优化的锚点。DPO 不是无脑提高 chosen 概率，而是希望当前模型相对 reference 更偏好 chosen，同时避免模型偏离原本语言分布太远。

**没有 reference 的风险：**

- 过拟合偏好数据。
- 输出风格变怪。
- 语言质量下降。
- 安全性变差。

## 38. GRPO 为什么最近常被问？

**面试答案：**  
GRPO 和 DeepSeek-R1、reasoning model 训练讨论相关。它对同一 prompt 采样一组回答，用组内相对奖励估计优势，减少对单独 value model 的依赖，从而降低 RL 训练复杂度。

**一句话：**  
GRPO 的核心是 group-relative advantage：同题多答案，组内比较，好的鼓励，差的压低。

## 39. 蒸馏在大模型里有什么用？

**面试答案：**  
蒸馏用强模型作为 teacher，把能力迁移到更小的 student 模型。它可以降低部署成本、提升小模型效果，也可以把 reasoning traces、偏好行为、领域能力迁移给小模型。

**常见方式：**

- response distillation：用大模型生成高质量回答训练小模型。
- logit distillation：让学生拟合教师概率分布。
- reasoning distillation：蒸馏思维链或推理轨迹。

## 40. 微调后模型幻觉变严重怎么办？

**面试答案：**  
先定位原因：数据是否有错误、是否过拟合、是否训练轮数过多、是否 loss mask 错、是否把不确定问题也训练成强答。解决上可以清洗数据、加入拒答样本、降低学习率/epoch、引入验证集早停、结合 RAG 提供证据、做人工 bad case 评估。

**项目话术：**  
我不会只看训练 loss 下降，还会看事实一致性、拒答能力、格式遵循和业务 bad case。幻觉问题通常不是靠继续训练解决，而是数据、检索、评估和安全策略一起做。
