# 公司面经雷达与 JD 关键词

更新时间：2026-06-19

这份文档用于每天看面经和 JD 时快速打标签。它不是“某公司一定问什么”的保证，而是从公开面经、题单、AI 应用开发攻略和岗位描述里提炼出的题型雷达。真实准备时，要以你投递的具体 JD、团队方向和简历项目为准。

## 怎么用

每天看 3-5 篇面经时，不要只摘题，按这个格式记录：

```text
公司/团队：
岗位：
轮次：
题目：
标签：Transformer / 训练对齐 / RAG / Agent / 推理部署 / 手撕 / 项目 / 安全 / 系统设计
考察层级：定义 / 公式 / 代码 / 排查 / 项目 / 上线
对应复习文件：
是否加入 101-140 追问：
```

判断优先级：

- 同一题在不同公司重复出现：升为红色。
- 和你简历项目强相关：升为红色。
- 只在一个岗位出现且和你方向不匹配：黄色或不记。
- 只是资料包标题党，没有真实题目：不记。

## 轮次雷达

一面常见：

- 基础八股：Attention、RoPE、KV Cache、LoRA、DPO、RAG。
- 手撕代码：MHA、Top-k/Top-p、RMSNorm、LoRA、DPO loss、PyTorch train loop。
- 项目 2 分钟版。

二面常见：

- 项目深挖：baseline、指标、bad case、为什么这样设计。
- 系统设计：RAG/Agent 怎么上线、怎么评估、怎么限流降级。
- 排查题：召回低、TTFT 高、工具调用失败、安全风险。

三面/主管面常见：

- 项目价值和业务理解。
- 取舍：效果、成本、延迟、安全怎么平衡。
- 你在项目里的真实贡献。
- 如果重做，怎么改。

HR/综合面常见：

- 为什么投这个方向。
- 为什么这家公司/团队。
- 最近关注的大模型热点。
- 实习/项目中最困难的事。

## 公司面经转错题的规则

只要出现下面任一情况，就加入错题：

- 题目能对应到 101-140，但你不能 2 分钟讲清。
- 题目直接追到你项目，但你没有指标或 bad case。
- 题目要求写代码，但你只能说思路。
- 题目问“为什么不用 X”，你只能说“因为大家都这么用”。
- 题目涉及安全/权限/灰度/回滚，但你项目话术里没有。

加入错题后写：

```text
原题：
公司/岗位：
我卡在哪里：
正确回答结构：
对应文件：
明天复述结果：绿 / 黄 / 红
```

## 一天只做三件事

看面经时很容易焦虑，所以每天只做三件事：

1. 记录 3 个高频题。
2. 把 1 个题补进自己的答案卡。
3. 把 1 个项目追问改成 2 分钟口语版。

不要把一天变成“收藏 50 篇面经”。资料越多，越要收敛。
