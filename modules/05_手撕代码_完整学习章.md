## 1. 手撕代码总原则

面试官看三件事：

- 你是否知道输入输出 shape。
- 你是否能写出核心逻辑。
- 你是否知道哪里容易错。

每题都按这个顺序讲：

```text
输入 shape -> 核心步骤 -> 输出 shape -> 复杂度 -> 易错点
```

## 2. Self-Attention

目标：

```text
x: [B, T, C]
out: [B, T, C]
```

核心：

```python
q = q_proj(x)
k = k_proj(x)
v = v_proj(x)
score = q @ k.transpose(-2, -1) / math.sqrt(d)
score = score.masked_fill(mask, float("-inf"))
attn = torch.softmax(score, dim=-1)
out = attn @ v
```

必须解释：

- score shape 是 `[B, T, T]`。
- softmax 在最后一维。
- causal mask 在 softmax 前。
- 缩放因子是 `sqrt(d)`。

## 3. Multi-Head Attention

shape：

```text
x:     [B, T, C]
qkv:   [B, T, 3C]
q/k/v: [B, H, T, D]
score: [B, H, T, T]
out:   [B, T, C]
```

伪代码：

```python
B, T, C = x.shape
H = num_heads
D = C // H

qkv = qkv_proj(x)                      # [B, T, 3C]
qkv = qkv.view(B, T, 3, H, D)
q, k, v = qkv.unbind(dim=2)             # [B, T, H, D]
q = q.transpose(1, 2)                   # [B, H, T, D]
k = k.transpose(1, 2)
v = v.transpose(1, 2)

score = q @ k.transpose(-2, -1) / math.sqrt(D)
score = score.masked_fill(mask, float("-inf"))
attn = torch.softmax(score, dim=-1)
out = attn @ v                          # [B, H, T, D]
out = out.transpose(1, 2).contiguous().view(B, T, C)
out = out_proj(out)
```

易错点：

- `C % H != 0`。
- mask 维度不能 broadcast。
- transpose 后直接 view。
- 缩放因子用了 `sqrt(C)`。

## 4. causal mask

```python
mask = torch.triu(torch.ones(T, T, device=x.device), diagonal=1).bool()
score = score.masked_fill(mask[None, None, :, :], float("-inf"))
```

解释：

- `diagonal=1` 表示主对角线右上角。
- 右上角是未来 token。
- mask 后 softmax，未来概率变 0。

## 5. RoPE

核心：

- 对 Q/K 做旋转。
- 每两个维度一组。
- cos/sin 根据 position 生成。

简化伪代码：

```python
def rotate_half(x):
    x1 = x[..., ::2]
    x2 = x[..., 1::2]
    return torch.stack((-x2, x1), dim=-1).flatten(-2)

def apply_rope(x, cos, sin):
    return x * cos + rotate_half(x) * sin
```

注意：

- 只作用在 Q/K。
- cos/sin shape 要能 broadcast 到 `[B, H, T, D]`。
- 偶奇维配对别写反。

## 6. RMSNorm

公式：

```text
y = x / sqrt(mean(x^2) + eps) * weight
```

代码：

```python
def rms_norm(x, weight, eps=1e-6):
    rms = torch.rsqrt(x.pow(2).mean(dim=-1, keepdim=True) + eps)
    return x * rms * weight
```

区别：

- RMSNorm 不减均值。
- LayerNorm 会减均值再除标准差。

## 7. Top-k Sampling

逻辑：

1. 找 logits 最大的 k 个。
2. 其他设为 `-inf`。
3. softmax。
4. multinomial 采样。

```python
def top_k_sample(logits, k):
    values, _ = torch.topk(logits, k)
    threshold = values[..., -1, None]
    filtered = logits.masked_fill(logits < threshold, float("-inf"))
    probs = torch.softmax(filtered, dim=-1)
    return torch.multinomial(probs, num_samples=1)
```

## 8. Top-p Sampling

逻辑：

1. 按概率降序排序。
2. 计算累计概率。
3. 保留累计概率达到 p 的最小集合。
4. 其他设为 `-inf`。

追问：

- Top-k 候选数固定。
- Top-p 候选数随分布变化，更动态。

## 9. BPE tokenizer 简化版

训练流程：

1. 初始化字符级词表。
2. 统计相邻 pair 频率。
3. 合并最高频 pair。
4. 重复直到词表达到目标大小。

面试表达：

> BPE 从字符级开始，不断合并高频子词，兼顾词表大小和 OOV 问题。

简化伪代码：

```python
for _ in range(num_merges):
    pairs = count_adjacent_pairs(corpus)
    best = max(pairs, key=pairs.get)
    corpus = merge_pair(corpus, best)
```

## 10. LoRA Linear

公式：

```text
out = xW^T + (xA^T)B^T * alpha / rank
```

代码：

```python
def lora_linear(x, weight, A, B, alpha, rank):
    base = x @ weight.T
    update = (x @ A.T) @ B.T
    return base + update * (alpha / rank)
```

解释：

- `weight` 冻结。
- A/B 可训练。
- rank 控制容量。

## 11. DPO Loss

输入：

- policy chosen logp。
- policy rejected logp。
- reference chosen logp。
- reference rejected logp。

代码：

```python
def dpo_loss(pc, pr, rc, rr, beta):
    pi_logratio = pc - pr
    ref_logratio = rc - rr
    return -F.logsigmoid(beta * (pi_logratio - ref_logratio)).mean()
```

解释：

> 让 policy 相对 reference 更偏好 chosen。

## 12. KV Cache 简化版

Prefill：

```python
k_cache = k
v_cache = v
```

Decode：

```python
k_cache = torch.cat([k_cache, k_new], dim=2)
v_cache = torch.cat([v_cache, v_new], dim=2)
out = attention(q_new, k_cache, v_cache)
```

shape：

```text
k_cache: [B, H_kv, T_cache, D]
v_cache: [B, H_kv, T_cache, D]
q_new:   [B, H_q, 1, D]
```

GQA/MQA 要处理 `H_q` 和 `H_kv` 不一致。

## 13. Beam Search

流程：

1. 保留 top beam_size 个候选序列。
2. 每步扩展每个候选。
3. 根据累计 log probability 选新的 top beam。
4. 直到 EOS 或最大长度。

追问：

- beam 越大越好吗？不一定，可能输出更保守、重复。
- 为什么用 log probability？避免概率连乘下溢，也方便累加。

## 14. AUC / NDCG

AUC：

> 随机抽一个正样本和一个负样本，模型给正样本打分高于负样本的概率。

NDCG：

> 排序任务指标，越相关的结果排在越前面分数越高，并用理想排序归一化。

适合推荐/搜索/广告相关岗位。

## 15. 2026 高频新增：GQA、序列 logprob、GRPO Advantage

### GQA repeat KV

Grouped Query Attention 里，Q 头数比 K/V 头数多：

```text
q: [B, H_q, T, D]
k: [B, H_kv, T, D]
v: [B, H_kv, T, D]
H_q = H_kv * n_rep
```

要把 K/V 复制到和 Q 头数匹配：

```python
def repeat_kv(x, n_rep):
    # x: [B, H_kv, T, D]
    if n_rep == 1:
        return x
    B, H_kv, T, D = x.shape
    x = x[:, :, None, :, :]              # [B, H_kv, 1, T, D]
    x = x.expand(B, H_kv, n_rep, T, D)   # [B, H_kv, n_rep, T, D]
    return x.reshape(B, H_kv * n_rep, T, D)
```

易错点：

- `expand` 不复制内存，后面用 `reshape` 得到目标视图/布局。
- `H_q` 必须能整除 `H_kv`。
- GQA 降低 KV Cache，但 Q 的计算头数仍是 `H_q`。

### Sequence Logprob

DPO、PPO、GRPO 都要算回答序列的 log probability。核心代码：

```python
import torch.nn.functional as F

def sequence_logprob(logits, labels, mask):
    # logits: [B, T, V]
    # labels: [B, T]
    # mask:   [B, T], answer token 为 1，prompt/pad 为 0
    log_probs = F.log_softmax(logits, dim=-1)
    token_logp = log_probs.gather(-1, labels.unsqueeze(-1)).squeeze(-1)
    return (token_logp * mask).sum(dim=-1)
```

如果 logits 是 next-token 预测，要注意 shift：

```python
shift_logits = logits[:, :-1, :]
shift_labels = labels[:, 1:]
shift_mask = mask[:, 1:]
```

易错点：

- 忘记 shift，导致拿当前位置预测当前位置。
- prompt、padding 没 mask。
- DPO chosen/rejected 长度差异大时，没有说明是 sum 还是 mean。

### GRPO Advantage

同一个 prompt 采样 `G` 个回答，reward shape 是 `[B, G]`：

```python
def grpo_advantage(rewards, eps=1e-6):
    mean = rewards.mean(dim=-1, keepdim=True)
    std = rewards.std(dim=-1, keepdim=True)
    return (rewards - mean) / (std + eps)
```

解释：

- 每个回答和同题其他回答比。
- 高于组内平均就是正 advantage。
- 低于组内平均就是负 advantage。
- 如果 reward 全一样，std 很小，要加 eps。

### 可运行小检查

面试里可以用一个很小的 sanity check 证明自己知道 shape：

```python
B, G = 2, 4
rewards = torch.tensor([[1.0, 0.0, 2.0, 1.0],
                        [3.0, 1.0, 1.0, 1.0]])
adv = grpo_advantage(rewards)
assert adv.shape == rewards.shape
assert torch.allclose(adv.mean(dim=-1), torch.zeros(B), atol=1e-6)
```

## 18. 面试前背诵版

手撕 LLM 代码时，最重要的是 shape。MHA 要从 `[B,T,C]` 讲到 `[B,H,T,D]`，mask 在 softmax 前，缩放因子是 `sqrt(head_dim)`。RoPE 作用在 Q/K，每两个维度旋转。RMSNorm 不减均值。LoRA 是冻结基座权重，只训练低秩 A/B。DPO loss 让 policy 相对 reference 更偏好 chosen。sequence logprob 要注意 next-token shift 和 answer mask。GRPO advantage 是同题多回答组内归一化。KV Cache 缓存历史 K/V，用显存换 decode 计算；GQA/MQA 通过减少 KV 头数降低缓存压力。
