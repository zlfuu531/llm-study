# 岗位模拟面试套卷

更新时间：2026-06-18

这份文件用于最后 2-3 周模拟面试。不要把它当题库通读，而是按目标岗位计时演练。每套控制在 45-60 分钟，答完后用评分表复盘。

## 使用方法

每轮模拟固定流程：

```text
3 min  自我介绍
15 min 项目深挖
15 min 技术八股 / 系统设计
15 min 手撕代码 / 公式推导
5 min  反问与复盘
```

评分：

| 项目 | 0 分 | 3 分 | 5 分 |
| --- | --- | --- | --- |
| 结构 | 答案散，听不出主线 | 有结构但细节弱 | 先结论，再机制，再项目落点 |
| 准确 | 概念错或混用 | 大体正确，有小漏洞 | 公式、shape、指标、边界都准确 |
| 深度 | 只能背定义 | 能解释常见追问 | 能讲 trade-off、排查链路、失败 case |
| 项目 | 只说用了工具 | 能讲方案和指标 | 能讲 baseline、ablation、bad case、上线 |
| 表达 | 卡顿、绕远 | 基本清楚 | 2 分钟内自然说完，能接追问 |

复盘规则：

- 单项低于 3 分，第二天优先补。
- 如果项目分低，回 [modules/06_项目面经与冲刺_完整学习章.md](modules/06_项目面经与冲刺_完整学习章.md)。
- 如果公式/指标分低，回 [06_公式与速查卡.md](06_公式与速查卡.md)。
- 如果追问压力不够，从 [03_高频题单100题.md](03_高频题单100题.md) 的 101-140 再抽 5 题。

## 套卷 A：大模型算法 / Post-training

适合岗位：大模型算法工程师、NLP 算法、模型微调/对齐、算法评测。

### A1 自我介绍追问

1. 你最近主要准备的大模型方向是什么？
2. 你最熟的一个模型相关项目是什么？
3. 如果只能选一个关键词证明你适合算法岗，你会选什么？

### A2 项目深挖

围绕你的微调、RAG 或模型实验项目追：

1. baseline 是什么，为什么不够？
2. 数据怎么来，怎么清洗，怎么划分 train/valid/test？
3. loss mask 怎么做，哪些 token 不算 loss？
4. 你做过哪些 ablation？哪一项提升最大？
5. 如果线上效果下降，你怎么判断是数据、模型还是分布漂移？
6. 如果用 Hugging Face 微调，你的数据、chat template、collator、Trainer 参数和 adapter 保存怎么设计？
7. 如果换 tokenizer 或扩词表，会影响哪些训练和推理环节？

### A3 技术八股

1. Self-Attention 公式和 shape。
2. 为什么除以 `sqrt(d_k)`？
3. LoRA 的低秩假设是什么？rank 怎么选？
4. QLoRA 的 NF4、double quantization、paged optimizer 分别解决什么？
5. DPO 的 loss 写一下，为什么需要 reference model？
6. GRPO 相比 PPO 省在哪里？
7. PPL 低为什么不等于模型真的好？
8. AdamW 为什么要解耦 weight decay？
9. SFTTrainer、DPOTrainer 和普通 Trainer 怎么区分？
10. BPE、SentencePiece 和 byte-level BPE 分别解决什么问题？
11. Temperature、Top-k、Top-p、Beam Search 分别怎么影响生成质量和稳定性？
12. GPTQ、AWQ、SmoothQuant 分别解决什么量化问题？

### A4 手撕 / 公式

二选一：

1. 手写 `sequence_logprob(logits, labels, mask)`，说明 next-token shift。
2. 手写 DPO loss，并解释 `beta` 过大过小的风险。

### A5 反问准备

可以问：

- 团队 post-training 主要做 SFT、偏好优化还是 reasoning RL？
- 评估集是公开 benchmark 为主，还是业务私有 eval 为主？
- 算法同学是否参与数据构造和线上 bad case 分析？

## 套卷 B：AI 应用开发 / RAG Agent

适合岗位：AI 应用开发工程师、Agent 工程师、大模型应用后端、RAG 工程。

### B1 自我介绍追问

1. 你做过的 RAG/Agent 项目解决了什么业务问题？
2. 你负责的是模型、检索、工具调用还是工程服务？
3. 你怎么证明项目不是 demo？

### B2 项目深挖

围绕 RAG/Agent 项目追：

1. 离线建库和在线问答链路完整说一遍。
2. chunk size 怎么定？overlap 为什么不是越大越好？
3. embedding 模型怎么选？为什么要 hybrid retrieval？
4. rerank 解决什么问题？如果 rerank 后更差怎么办？
5. Agent 为什么不用固定 workflow？
6. 工具调用参数错了怎么办？
7. 怎么防止 prompt injection 和工具越权？
8. 你如何构造私有 eval？
9. 结构化输出或工具调用时，解码参数和 constrained decoding 怎么选？

### B3 技术八股 / 系统设计

1. RAG 召回率低，从哪些层排查？
2. faithfulness、context precision、Recall@k 分别看什么？
3. Agent Harness 包括哪些模块？
4. MCP 的 Host、Client、Server、Tools、Resources、Prompts 分别是什么？
5. Human-in-the-loop 放在哪些环节？
6. 记忆系统怎么设计，怎么避免污染和泄露？
7. 模型网关需要做哪些事？
8. RAG/Agent 的 token budget 怎么拆？超预算时先裁哪里？
9. 如果回答重复、跑偏或格式不稳定，你怎么从 prompt、上下文和 decoding 参数排查？
10. 长上下文模型为什么仍然需要 RAG 和 Context Engineering？
11. 如果面试官追 RoPE scaling、YaRN 或 Ring Attention，你怎么把它们和应用系统联系起来？

### B4 手撕 / 伪代码

二选一：

1. 写一个工具调用前的参数校验伪代码：schema、权限、风险确认、重试上限。
2. 写一个 RAG bad case 排查函数：输入 query、gold docs、retrieved chunks、answer，输出问题层级。
3. 写一个长上下文 eval 分桶伪代码：长度、证据位置、多跳、干扰、无答案、引用正确率。

### B5 反问准备

可以问：

- 团队现在 RAG/Agent 的主要瓶颈是效果、延迟、成本还是安全？
- 是否有固定的私有 eval 和线上 bad case 回流流程？
- AI 应用开发岗需要参与模型选型和模型网关设计吗？

## 套卷 C：推理部署 / AI Infra

适合岗位：推理部署、模型服务、AI 平台、LLM Serving、Infra。

### C1 自我介绍追问

1. 你了解哪些推理框架或服务化方案？
2. 你做过哪些延迟、吞吐、显存或成本优化？
3. 你怎么理解线上大模型服务的稳定性？

### C2 项目深挖

围绕推理/服务项目追：

1. prefill 和 decode 的瓶颈分别是什么？
2. TTFT 和 TPOT 分别受哪些因素影响？
3. KV Cache 显存怎么估算？
4. PagedAttention 和 FlashAttention 区别？
5. Continuous Batching 为什么能提高吞吐？
6. Prefix Cache 什么时候有效？
7. Chunked Prefill 解决什么问题？
8. 线上高峰期怎么限流、降级和熔断？
9. INT4 量化为什么不一定比 FP16 更快？
10. KV Cache 量化有什么收益和质量风险？
11. Speculative decoding 为什么能无损加速？accept rate 低时为什么可能变慢？
12. EAGLE、Medusa、MTP 和普通 draft model 有什么区别？
13. Prefill/Decode disaggregation 为什么出现？KV Cache 传输难在哪里？
14. TP、PP、DP、EP 在推理 serving 里分别解决什么？

### C3 系统设计

设计一个大模型在线推理服务：

必须覆盖：

- API gateway / model gateway。
- 鉴权、租户隔离、token quota。
- 模型路由：大模型、小模型、量化模型。
- 请求队列和 batching。
- KV Cache、prefix cache。
- 量化策略：FP16 baseline、INT8/W8A8、GPTQ/AWQ INT4、FP8/KV Cache quant。
- 推测解码：draft model / EAGLE / MTP / n-gram，按 accept rate 和负载动态开关。
- 分布式推理：PD 分离、KV transfer、TP/PP/EP、多副本、MoE serving。
- 流式输出。
- 监控：TTFT、TPOT、tokens/s、P95/P99、错误率、成本。
- 灰度和回滚。

### C4 手撕 / 公式

三选一：

1. 写 KV Cache 显存估算公式，并代入一个 7B 模型说明哪些变量最敏感。
2. 写一个简化的 top-k/top-p sampling，说明温度和候选集合的影响。
3. 手写一个 PyTorch 训练循环，覆盖 AMP、gradient accumulation、clip、scheduler、eval 和 checkpoint 保存点。
4. 写对称/非对称量化公式，并说明 per-channel 和 group-wise 的区别。
5. 写 speculative sampling 的接受概率 `min(1,p/q)`，说明拒绝后为什么要修正采样。
6. 画 PD disaggregation 链路，并标出 TTFT、TPOT、KV transfer 和队列指标。

### C5 反问准备

可以问：

- 团队使用 vLLM、TensorRT-LLM、SGLang 还是自研 serving？
- 线上优化更看重 TTFT、TPOT、吞吐还是成本？
- 是否有多模型路由、量化和 prefix cache 的线上实践？

## 套卷 D：搜广推 + LLM / 算法应用

适合岗位：搜索、推荐、广告、风控、内容理解中的 LLM 方向。

配套复习：[32_搜索推荐广告_LLM_Ranking与LTR面试.md](32_搜索推荐广告_LLM_Ranking与LTR面试.md)、[answers/27_搜索推荐广告_LLM_Ranking与LTR_答案版.md](answers/27_搜索推荐广告_LLM_Ranking与LTR_答案版.md)

### D1 自我介绍追问

1. 你是否做过排序、召回、推荐或评估指标相关项目？
2. LLM 在你的项目里是生成、rerank、特征增强还是问答？
3. 你怎么把模型指标和业务指标连起来？

### D2 项目深挖

1. AUC、NDCG、MRR、Recall@k 分别适合什么任务？
2. 线上 AUC 提升但业务不涨，可能为什么？
3. LLM rerank 会带来什么延迟和成本问题？
4. 如何把 RAG 的检索评估和搜索/推荐指标打通？
5. 类别不平衡时为什么 PR-AUC 可能比 ROC-AUC 更有意义？
6. query rewrite 或语义召回如何做离线评估？

### D3 技术八股

1. AUC 的概率解释。
2. NDCG 为什么要折损位置？

5. 双塔召回和 cross-encoder rerank 的速度/效果取舍。
6. RAG 的 hybrid retrieval 和传统多路召回有什么相似点？

8. LambdaMART 为什么和 NDCG 联系很紧？
9. 位置偏差和曝光偏差怎么处理？

### D4 手撕 / 计算

二选一：

1. 手写 AUC 的排序计算思路。
2. 给一组相关性分数，手算 DCG / IDCG / NDCG。

### D5 反问准备

可以问：

- 团队里 LLM 更多用于召回、排序、生成还是评估？
- 线上实验更关注哪些业务指标？
- LLM 带来的延迟成本如何被纳入排序系统？

### E1 自我介绍追问

1. 你做过图文、OCR、视频或多模态 Agent 项目吗？
2. 你怎么理解视觉 encoder 接到 LLM？
3. 你熟悉哪些多模态评估指标或 bad case？

### E2 项目深挖

1. 图像 token 怎么接入 LLM hidden space？
2. projector、adapter、Q-Former 大致解决什么？
3. OCR 细粒度识别为什么难？

8. VLM 输出 bbox 或 JSON 后，你怎么证明它真的可靠？

### E3 技术八股

1. Encoder-only、Decoder-only、Encoder-Decoder 怎么选？
2. CLIP 式对比学习的直觉是什么？
3. 图像 patch/token 和文本 token 的差异是什么？
4. 多模态数据清洗有哪些特殊问题？
5. 如果 VLM 在 OCR 上错，可能是 encoder、resolution、tokenizer 还是数据问题？
6. 视频理解为什么不是把多帧图片直接拼进上下文？
7. GUI Agent 里 OSWorld、ScreenSpot、UI-TARS、OmniParser 分别考察什么？

### E4 手撕 / 伪代码

二选一：

1. 写一个简化的 InfoNCE / 对比学习 loss。

### E5 反问准备

可以问：

- 团队多模态任务更偏 OCR、图文问答、视频理解还是 Agent？
- 是否有专门的多模态 eval 和人工标注流程？
- 多模态线上部署主要瓶颈是延迟、显存还是效果？
- GUI/文档/视频场景里是否要求 bbox、页码、时间戳或动作 trace 作为证据？

## 模拟面试禁忌

- 不要每题都讲 5 分钟，真实面试需要先给结论。
