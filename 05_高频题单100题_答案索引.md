## 101-140 追问题复习法

每题按这个顺序过：

1. 先 30 秒说结论。
2. 再 2 分钟讲排查链路或公式。
3. 最后补一个项目例子或线上风险。

### Tokenizer / Token 预算

- Tokenizer 是模型输入协议，不只是分词
- BPE：高频 pair merge；编码按 merge rank 合并

### 解码策略 / Sampling

- 生成 loop：logits -> processor/warper -> next token -> KV Cache
- Temperature：`softmax(logits / T)`
- Greedy 稳定但保守，sampling 多样但不稳定
- Top-k 固定候选数量，Top-p 控制累计概率质量
- Beam Search 保留多条高分路径，适合目标确定任务
- Length penalty 缓解 beam 短序列偏置
- Repetition / presence / frequency penalty 不是质量万能开关
- LogitsProcessor：bad words、min length、no-repeat、forced tokens
- Constrained decoding 保证语法/schema，不保证业务正确
- EOS / stop strings / max_new_tokens / seed / generation config 要一起版本化

### 模型量化 / 低比特推理

- 对称量化：`q=round(x/s)`，`x_hat=s*q`
- 非对称量化：`q=round(x/s+z)`，`x_hat=s*(q-z)`
- FP16 权重约 `2P bytes`，INT8 约 `1P bytes`，INT4 主体约 `0.5P bytes`
- PTQ 成本低先上线，QAT 质量更稳但训练成本高
- weight-only 省权重显存，W8A8 要解决激活 outlier，KV quant 省长上下文缓存
- per-tensor 简单但粗，per-channel 更准，group-wise 是 LLM 低比特常用折中
- GPTQ：近似二阶误差补偿
- AWQ：activation-aware，保护敏感通道
- SmoothQuant：平滑激活 outlier，把难度迁移到权重
- QLoRA：4-bit 冻结基座 + LoRA，NF4 / double quant / paged optimizer
- FP8：浮点低精度，依赖硬件、scaling 和 Transformer Engine 等生态
- INT4 不一定快：dequant、metadata、kernel、硬件、batch shape 和端到端瓶颈都会影响

### 推测解码 / Speculative Decoding

- 基本流程：draft proposes multiple tokens -> target verifies -> accept prefix
- sampling 接受概率：`a(y)=min(1, p(y)/q(y))`
- 拒绝后修正分布：`r(x) ∝ max(0, p(x)-q(x))`
- 加速条件：draft 便宜、target 验证并行、accept rate 高、瓶颈在 decode
- 关键指标：accept rate、mean accepted length、TPOT/ITL、tokens/s、P95/P99、显存
- Draft model：便宜且像 target，tokenizer/vocab 最好一致
- n-gram/suffix：从上下文重复片段草拟，便宜但适用面窄
- Medusa：多个 decoding heads + tree attention
- EAGLE：feature-level speculation
- MTP：多 token 预测模块/目标，可作为原生 proposer
- 可能变慢：低接受率、draft 太重、tree 太大、高 QPS、大 batch、prefill 是瓶颈、guided decoding 冲突

### 分布式推理 / PD 分离 / MoE Serving

- Prefill：处理完整 prompt，偏算力和大矩阵，主要影响 TTFT
- Decode：逐 token 生成，偏 KV Cache 读写、HBM 和调度，主要影响 TPOT/ITL
- Aggregated serving：同一池 GPU 同时跑 prefill/decode，简单但会互相干扰
- PD disaggregation：prefill pool + KV transfer + decode pool，能独立扩缩容
- KV 显存：`B * S * L * H_kv * D * 2 * bytes`
- KV transfer 难点：KV 大、layout/page 映射、TP size 不同、预分配、取消和清理
- TP：切层内矩阵；PP：切层；DP/replica：扩 QPS；EP：切 MoE experts
- Chunked prefill 是切小 prefill，PD 分离是分资源池，prefix cache 是复用前缀 KV
- 路由看：prompt length、max_new_tokens、prefill/decode queue、KV capacity、SLO、transfer bandwidth
- MoE serving 难点：router、token dispatch、all-to-all、hot experts、load imbalance、tail latency
- PD 不一定划算：短 prompt、低流量、网络差、KV transfer 慢、prefix cache 已解决大部分 prefill
