# LLM学习

更新时间：2026-06-20

这是一套面向所有 LLM 学习者的结构化资料。它把大模型核心知识、常见问答、手撕代码、RAG/Agent、工程实践、项目复盘、面试题单和优质学习网站整理成一个可以直接阅读、搜索和复习的学习入口。

## 入口

- 在线阅读：[GitHub Pages](https://zlfuu531.github.io/llm-study/)
- 源资料仓库：[zlfuu531/llm-study](https://github.com/zlfuu531/llm-study)
- 本地离线阅读：打开当前文件夹里的 `LLM学习.html`
- 本地重新生成单页：

```powershell
node .\tools\build-learning-html.mjs
```

说明：线上入口是 `index.html`，它已经内置全部 Markdown 内容。GitHub 仓库同时保留 Markdown 源文件，方便直接查看、搜索、引用和后续继续整理。

## 推荐阅读顺序

1. 先读 [LLM 学习路线与阶段性冲刺](00_两个月冲刺总路线.md)，确定自己的学习主线。
2. 再读 [每日学习法](02_每日学习法.md)，把每天的阅读、复述、手撕和复盘固定下来。
3. 按 `modules/` 里的完整学习章推进，优先学 Transformer、训练微调对齐、RAG/Agent/MCP、推理部署和手撕代码。
4. 需要问答训练时，刷 [高频题单 100 题](03_高频题单100题.md) 和 [答案索引](05_高频题单100题_答案索引.md)。
5. 需要快速回忆公式、指标和工程排查时，看 [公式与速查卡](06_公式与速查卡.md)。
6. 想继续找原始资料时，看 [资源优先级与口碑来源](01_资源优先级与口碑来源.md)、[外部资料本地索引](07_外部资料本地索引.md) 和 [原始资料下载清单](04_原始资料下载清单.md)。

## 资料分层

- `modules/`：完整学习章，适合一章一章系统读。
- `chapters/`：更短的专题章节，适合快速补概念。
- `deepdives/`：深挖专题，适合把公式、shape、复杂度和工程联系讲透。
- `answers/`：问答答案版，适合模拟面试、复述和查漏补缺。
- `templates/`：答案卡片、项目复盘、面经复盘和每日计划模板。
- 根目录专题文件：系统设计、多模态、安全、端侧、数据工程、评测、训练系统、推理引擎、GPU/CUDA、开源生态、Prompt、Tokenizer、长上下文、数据合成等高频方向。

## 学习原则

- 每个知识点都按“定义 -> 为什么需要 -> 怎么做 -> 优缺点 -> 项目里怎么用 -> 可能追问”来准备。
- 不只背答案，先自己写 3-5 句话，再对照答案版修改。
- 手撕代码要每天练，优先 Attention、RoPE、RMSNorm、LoRA、DPO、Top-k/Top-p、KV Cache 等 LLM 高频模块。
- 面经只用来反推重点：记录重复出现、自己答不出、和项目相关的问题。
- 项目表达比资料收藏更重要。每个项目都要能说清背景、方案、指标、难点、复盘和改进。
