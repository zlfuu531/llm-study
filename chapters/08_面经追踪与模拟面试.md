# 面经追踪与模拟面试

## 面经怎么用

面经不是拿来制造焦虑的，而是用来校准复习重点。

只记录三类问题：

- 高频重复。
- 自己不会。
- 和自己项目相关。

不要记录：

- 情绪发泄。
- 标题党资料包。
- 明显不匹配岗位的问题。

## 每日面经流程

1. 找 3-5 篇 2025-2026 面经。
2. 标出岗位：算法、应用开发、平台、推理、研究。
3. 摘出问题。
4. 给问题打标签。
5. 判断自己是否能 2 分钟回答。
6. 不能回答就加入错题库。

标签：

- Transformer
- 微调
- 对齐
- RAG
- Agent
- 推理部署
- 手撕代码
- 项目追问
- 编程基础
- 数学/机器学习

## 高频面试组合

### 算法岗

常见组合：

- Transformer 细节
- 训练/微调/对齐
- 数学和机器学习基础
- 手撕 Attention/RoPE/LoRA
- 简历项目深挖

重点：

- 公式和代码。
- 数据和实验。
- paper 细节。

### 大模型应用开发岗

常见组合：

- RAG
- Agent
- 工具调用
- Prompt
- 系统设计
- 项目落地和评估

重点：

- 工程链路。
- 失败处理。
- 评估指标。
- 成本和延迟。

### 推理部署岗

常见组合：

- KV Cache
- vLLM
- PagedAttention
- 量化
- batch 调度
- GPU 显存
- 并发和延迟

重点：

- 系统性能。
- 资源估算。
- 工程取舍。

## 模拟面试结构

总长 45-60 分钟。

1. 自我介绍：3 分钟。
2. 项目深挖：15-20 分钟。
3. LLM 八股：15 分钟。
4. 手撕代码：15-20 分钟。
5. 反问：3 分钟。

## 自我介绍模板

```text
面试官您好，我叫 X，目前主要方向是大模型相关的算法/应用开发。
我比较熟悉 Transformer、RAG、Agent、微调和推理部署这些内容。
项目上，我做过 X 项目，主要解决 Y 问题，我负责 Z 部分，里面涉及到 A、B、C 技术。
最近我重点学习的是大模型相关方向，所以也系统复习了手撕 Attention、RoPE、LoRA、DPO 以及 RAG/Agent 的工程评估。
```

注意：

- 不要超过 3 分钟。
- 不要罗列所有课程和工具。
- 引导面试官问你准备好的项目。

## 八股模拟题

### Transformer

1. Self-Attention 公式是什么？
2. 为什么除以 `sqrt(d_k)`？
3. MHA 和 GQA 区别？
4. KV Cache 怎么节省计算？
5. RoPE 作用在哪些张量上？

### 微调对齐

1. SFT 和预训练区别？
2. LoRA 为什么省参数？
3. QLoRA 为什么省显存？
4. RLHF 流程？
5. DPO 和 PPO 区别？

### RAG

1. RAG 全链路是什么？
2. chunk size 怎么选？
3. 为什么要 rerank？
4. RAG 幻觉怎么解决？
5. 怎么评估 RAG？

### Agent

1. Agent 和 Chatbot 区别？
2. ReAct 流程？
3. Function Calling 本质？
4. MCP 解决什么问题？
5. Agent 死循环怎么办？

### 推理部署

1. prefill 和 decode 区别？
2. TTFT 和 TPOT 是什么？
3. PagedAttention 解决什么问题？
4. 量化有什么风险？
5. 高并发下怎么优化？

## 面试后复盘

马上写：

- 哪些题答得好？
- 哪些题答得虚？
- 哪个项目追问没接住？
- 手撕代码卡在哪里？
- 哪个概念需要重新学？
- 下次应该主动引导到哪个项目？

## 复盘评分

每场面试给自己打分：

- 项目表达：0-5
- 八股准确度：0-5
- 手撕代码：0-5
- 追问承压：0-5
- 沟通流畅：0-5

低于 3 分的项，第二天优先补。
