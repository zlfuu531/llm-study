# 项目复盘与简历话术

## 面试目标

项目面不是背项目介绍，而是证明你真正做过、理解取舍、能复盘问题。

你需要准备：

- 2 分钟项目介绍。
- 8 分钟项目深挖。
- 10 个追问题。
- 3 个失败 case。
- 3 个指标。
- 1 个如果重做会怎么改。

## 项目讲法结构

推荐结构：

1. 背景：为什么要做。
2. 目标：解决什么问题。
3. 方案：系统怎么设计。
4. 你的工作：你负责什么。
5. 难点：遇到什么问题。
6. 指标：效果如何。
7. 复盘：还能怎么改。

## RAG 项目话术

2 分钟版：

> 我做的是一个面向 X 场景的知识库问答系统。离线侧会对文档做解析、清洗和 chunk 切分，用 embedding 建向量索引，同时保留关键词检索做混合召回。在线侧用户问题先做 query rewrite，然后召回候选文档，再用 reranker 排序，最后把 top 文档拼进 prompt 让模型基于证据回答，并返回引用来源。我主要负责 X、Y、Z。项目里比较关键的问题是 chunk 粒度、专有名词召回和幻觉控制，最后我们用 Recall@k、答案一致性、人工满意度和延迟成本做评估。

追问准备：

- 为什么不用微调？
- chunk size 怎么选？
- embedding 模型怎么选？
- 为什么要 rerank？
- 怎么评估答案是否基于证据？
- 文档更新怎么处理？
- 如果用户问的问题跨多个文档怎么办？
- 如果召回为空怎么办？

## Agent 项目话术

2 分钟版：

> 我做的是一个能调用工具完成 X 任务的 Agent。系统分为任务理解、规划、工具调用、状态管理和结果生成几部分。模型根据用户目标生成计划，再按工具 schema 调用检索、数据库或业务 API。每次工具返回后进入 observation，模型判断下一步。为了避免不稳定，我加了参数校验、最大步数、失败重试、权限控制和 trace 日志。评估上主要看任务完成率、工具调用正确率、平均步骤数、失败率和成本。

追问准备：

- Agent 和普通工作流有什么区别？
- 为什么要让模型规划，而不是写死流程？
- 怎么避免死循环？
- 工具调用失败怎么办？
- 怎么做权限控制？
- 怎么评估工具调用是否正确？
- 记忆怎么设计？

## 微调项目话术

2 分钟版：

> 我做的是一个面向 X 任务的模型微调项目。首先判断 prompt/RAG 是否能解决，确认需要领域适配后，我们收集并清洗了 X 类数据，统一成 instruction-response 格式，对敏感信息做脱敏，并划分训练、验证和测试集。训练上选择 LoRA/QLoRA，冻结基座模型，只训练低秩适配参数。评估上除了自动指标，还做了人工 case 评估和业务指标对比。主要难点是数据质量、过拟合和模型在边界问题上的幻觉。

追问准备：

- 为什么不用全量微调？
- LoRA rank 怎么选？
- 训练数据怎么清洗？
- 如何防止数据泄露？
- 如何评估过拟合？
- 微调后安全性变差怎么办？

## 推理部署项目话术

2 分钟版：

> 我参与的是一个大模型推理服务优化项目。系统需要支持多用户并发和流式输出，所以我们重点优化了模型加载、请求调度、KV Cache、批处理和监控。指标上看 TTFT、TPOT、tokens/s、P95 延迟和 GPU 利用率。优化手段包括量化、prefix cache、continuous batching、合理限制上下文长度和超时降级。主要难点是吞吐和延迟的平衡，以及长上下文带来的 KV Cache 显存压力。

追问准备：

- prefill 和 decode 哪个更慢？
- 量化后效果怎么评估？
- P95 延迟高怎么排查？
- 长上下文显存不够怎么办？
- 流式输出怎么实现？
- 并发上来后如何限流？

## 指标怎么讲

不要只说“效果不错”。

可讲指标：

- 准确率、召回率、F1。
- Recall@k、NDCG、MRR。
- 人工满意度。
- 幻觉率。
- 解决率、转人工率。
- TTFT、TPOT、P95 延迟。
- tokens/s、QPS。
- 单次调用成本。

如果没有真实线上指标：

- 说明你用的是离线评测集。
- 说明评测集怎么构造。
- 说明有哪些 case 仍然失败。

## 项目最容易被追问穿的地方

- 只说用了 LangChain，不知道链路。
- 只说用了向量数据库，不知道 chunk 和 embedding 怎么选。
- 只说微调了模型，不知道 loss mask 和数据格式。
- 只说提升了效果，没有指标。
- 只说部署了模型，不知道显存估算和延迟指标。
- 只说 Agent 能调用工具，不知道失败和权限怎么处理。

## 准备清单

每个项目写一页：

- 项目一句话。
- 业务背景。
- 技术架构。
- 你负责的部分。
- 三个难点。
- 三个指标。
- 三个 bad case。
- 五个替代方案。
- 十个追问题。
- 如果重做会怎么改。

## 反问问题

可以准备：

- 团队目前大模型应用主要在哪些场景落地？
- 面试岗位更偏模型训练、应用开发还是推理部署？
- 团队如何评估 RAG/Agent 系统效果？
- 线上大模型服务最主要的瓶颈是延迟、成本还是效果？

不要问：

- 能不能少加班。
- 有没有培训。
- 这个岗位是不是很难。
- 薪资多少，除非到 HR 阶段。
