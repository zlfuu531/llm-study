# 训练、微调与对齐

## 面试目标

这一章要达到的状态：

- 能讲清预训练、SFT、RLHF、DPO 的区别。
- 能解释 LoRA / QLoRA 的核心思路。
- 能回答“给你业务数据，你怎么微调一个模型”。

## 训练阶段总览

大模型训练通常可以粗略分成：

1. 预训练：让模型学语言和世界知识。
2. 指令微调 SFT：让模型学会按指令回答。
3. 偏好对齐：让模型更符合人类偏好和安全要求。
4. 领域适配：面向业务场景继续微调或 RAG。

面试表达：

> 预训练解决“会不会语言和知识”，SFT 解决“会不会按指令回答”，偏好对齐解决“回答是否更符合人类偏好”，领域适配解决“在具体业务场景下是否可用”。

## 预训练

Decoder-only LLM 常用 Causal Language Modeling：

```text
给定 x_1...x_t，预测 x_{t+1}
```

loss：

```text
cross entropy over next token
```

常见追问：

- 为什么 GPT 用 causal LM？
- BERT 的 masked LM 和 GPT 的 causal LM 有什么区别？
- 数据质量为什么比数据量更关键？
- 训练时 teacher forcing 是什么？

回答要点：

- GPT 生成任务天然要求从左到右预测。
- causal mask 保证不能看未来。
- 训练时所有位置并行计算 loss，推理时逐 token 生成。

## SFT

SFT，即 Supervised Fine-Tuning。用人工构造或清洗的指令数据训练模型，让模型学会指令跟随。

数据格式：

- instruction
- input 或 context
- response

要点：

- 训练目标仍然通常是 next token prediction。
- 一般只对 answer 部分计算 loss，避免模型学习复述 prompt。
- 数据质量、格式一致性和多样性很重要。

面试表达：

> SFT 的本质还是监督学习，只是数据从普通语料变成了指令-回答对。它让预训练模型从“续写文本”转向“按用户指令完成任务”。

## LoRA

LoRA，即 Low-Rank Adaptation。

核心思想：

- 冻结原模型权重 `W`。
- 增加低秩矩阵 `A` 和 `B`。
- 更新量 `ΔW = BA`。
- 推理时可以合并到原权重。

公式：

```text
y = Wx + scale * B(Ax)
```

为什么有效：

- 全量微调参数量太大。
- 很多任务的权重更新可能具有低秩结构。
- 只训练小矩阵，显存和存储成本低。

常见追问：

- LoRA 通常加在哪些层？
- rank 怎么选？
- alpha 的作用是什么？
- LoRA 和 Adapter / Prefix Tuning 的区别？
- LoRA 能不能合并权重？

## QLoRA

QLoRA 是在量化基座模型上做 LoRA 微调。

要点：

- 基座模型通常 4-bit 量化。
- LoRA 参数保持较高精度训练。
- 通过降低基座权重显存，让单卡微调更可行。

面试表达：

> QLoRA 的核心不是把 LoRA 也量化到 4-bit，而是把冻结的基座模型量化存储，同时训练较小的 LoRA 参数，从而显著降低显存。

## RLHF

RLHF 常见流程：

1. SFT 得到初始策略模型。
2. 收集人类偏好数据，例如同一 prompt 的两个回答谁更好。
3. 训练 Reward Model。
4. 用 PPO 等强化学习方法优化策略模型。

优点：

- 能利用人类偏好。
- 比单纯 SFT 更适合对齐复杂偏好。

缺点：

- 流程复杂。
- 训练不稳定。
- Reward hacking 风险。
- 标注成本高。

## PPO

PPO 是一种限制策略更新幅度的强化学习算法。

面试不要陷入完整推导，重点讲：

- 策略模型生成回答。
- Reward Model 给奖励。
- PPO 优化策略，同时用 KL 约束避免偏离原模型太远。

常见追问：

- 为什么需要 KL penalty？
- PPO 为什么训练复杂？
- RLHF 和 SFT 的区别？

## DPO

DPO，即 Direct Preference Optimization。

核心：

- 不显式训练 Reward Model。
- 直接用偏好对优化模型。
- 形式上让 chosen response 的概率相对 rejected response 更高。

面试表达：

> DPO 把偏好学习转成一个更简单的监督式目标，不需要显式训练 reward model，也不需要像 PPO 那样做在线强化学习。它直接利用 chosen/rejected 的 log probability 差异，让模型更偏向人类喜欢的回答。

常见追问：

- DPO 和 PPO 的区别？
- DPO 为什么还需要 reference model？
- beta 控制什么？
- 偏好数据质量差会怎样？

## GRPO

GRPO 常和 DeepSeek 系模型一起被问。

核心记忆：

- 它是 group relative 的思想。
- 对同一个 prompt 采样多个回答。
- 用组内相对表现来估计优势。
- 目标是减少对单独 critic/value model 的依赖。

面试表达：

> GRPO 的直觉是同一个问题生成一组回答，用组内相对奖励来判断哪些回答更好，从而做策略优化。它的关注点是降低 RL 训练复杂度和资源消耗，常被放在 DeepSeek-R1 相关上下文里讨论。

## 如果给你业务数据，怎么微调

回答结构：

1. 明确任务：问答、分类、抽取、代码、客服、检索增强。
2. 判断是否需要微调：能否用 prompt/RAG 解决。
3. 数据处理：去重、脱敏、格式统一、质量过滤。
4. 划分数据：train/valid/test。
5. 选择方案：SFT、LoRA、QLoRA、DPO。
6. 训练配置：学习率、batch、epoch、max length、loss mask。
7. 评估：自动指标 + 人工评估 + 业务指标。
8. 部署：量化、服务、灰度、监控。
9. 风险：幻觉、安全、数据泄露、过拟合。
