# 手撕代码训练营

## 训练原则

手撕代码的目标不是背完整工程实现，而是在面试现场写出核心逻辑，并能解释 shape、复杂度和边界情况。

每天流程：

1. 看题 2 分钟，写 shape。
2. 独立写代码 20-30 分钟。
3. 跑最小样例。
4. 记录 bug。
5. 关掉参考再写一次。

## MHA 训练卡

必须能写出的步骤：

1. 输入 `x: [B, T, C]`
2. `qkv_proj(x) -> [B, T, 3C]`
3. reshape 成 `[B, T, 3, H, D]`
4. permute 成 `q/k/v: [B, H, T, D]`
5. score = `q @ k.transpose(-2, -1) / sqrt(D)`
6. 加 causal mask
7. softmax + dropout
8. `attn @ v -> [B, H, T, D]`
9. transpose + reshape 回 `[B, T, C]`
10. output projection

常见 bug：

- `C % H != 0` 没检查。
- mask 维度错。
- `view` 前没有 `contiguous()`。
- softmax 维度写错，应该在最后一维。
- 缩放因子用了 `sqrt(C)` 而不是 `sqrt(D)`。

## RoPE 训练卡

必须能解释：

- RoPE 作用在 Q/K。
- 每两个维度一组做旋转。
- 旋转角度和 position、frequency 有关。
- 内积后能反映相对位置信息。

实现步骤：

1. 生成频率 `inv_freq`。
2. 根据 position 得到角度。
3. 计算 cos/sin。
4. 把偶数维和奇数维配对旋转。
5. 应用到 q/k。

常见 bug：

- cos/sin shape 不能 broadcast。
- 旋转偶奇维顺序写反。
- 把 RoPE 加到了 V 上。

## RMSNorm 训练卡

公式：

```text
y = x / sqrt(mean(x^2) + eps) * weight
```

要点：

- 不减均值。
- 沿 hidden dimension 计算。
- `eps` 防止除零。

## Top-k / Top-p 训练卡

Top-k：

- 只保留概率最高的 k 个 token。
- 其他 token 置为 `-inf`。
- 再 softmax 采样。

Top-p：

- 按概率从高到低排序。
- 保留累计概率达到 p 的最小集合。
- 其他 token 置为 `-inf`。

常见追问：

- Top-k 和 Top-p 谁更动态？
- temperature 作用在哪里？
- 为什么采样前要处理 logits？

## BPE 训练卡

核心流程：

1. 初始化字符级词表。
2. 统计相邻 pair 频率。
3. 合并最高频 pair。
4. 重复直到达到词表大小。

面试重点：

- BPE 是从小粒度逐步合并。
- 能缓解 OOV。
- 中文、代码、多语言场景会有不同 tokenizer 选择。

## KV Cache 训练卡

简化实现：

- 第一次 prefill 时保存所有 K/V。
- decode 时把新 K/V concat 到 cache。
- 新 token 的 Q attend 到全部 K/V。

要点：

- cache 维度通常是 `[B, H_kv, T_cache, D]`。
- GQA/MQA 时 Q heads 和 KV heads 数量不同，需要 repeat 或 group。

## LoRA 训练卡

核心：

```text
out = base_linear(x) + scale * B(A(x))
```

要点：

- base weight 冻结。
- A/B 是低秩矩阵。
- `scale = alpha / rank`。
- 推理时可以 merge。

## DPO Loss 训练卡

输入：

- prompt
- chosen response
- rejected response
- policy model logp
- reference model logp

核心：

```text
pi_logratio = logp_policy_chosen - logp_policy_rejected
ref_logratio = logp_ref_chosen - logp_ref_rejected
loss = -logsigmoid(beta * (pi_logratio - ref_logratio))
```

要点：

- chosen 应该比 rejected 概率更高。
- reference model 防止模型偏离太远。
- beta 控制偏好强度。
