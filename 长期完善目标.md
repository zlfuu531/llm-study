# 长期完善目标：LLM学习

更新时间：2026-06-18

这个站点的目标不是“资料越多越好”，而是持续吸收市面上高质量 LLM 学习网站、问答题库、面试经验、项目实践、GitHub 仓库、官方文档和论文里的分类方式与高频内容，把它们沉淀成所有学习者都能直接阅读、复习、追问和项目表达的材料。

## 总目标

持续学习外部资料，并不断完善 `modules/` 里的完整学习章，让每章都达到下面这个标准：

- 能从基础讲起，不默认你已经完全懂。
- 有通俗直觉，也有公式、张量 shape、复杂度和工程含义。
- 有高频面试问答、追问链路、常见误区和可背诵版本。
- 能连接到项目经验：为什么这样设计、指标怎么评估、线上会遇到什么坑。
- 对 2025-2026 的新热点保持更新，但不追噱头，优先整理面试真的会问、项目真的能用的部分。

## 每日 heartbeat 要做什么

每日自动完善时，按这个顺序推进：

1. 调研当天或近期高频来源：牛客面经、小红书/知乎/B站攻略、GitHub 八股仓库、课程笔记、官方文档和代表论文。
2. 提取“分类体系”而不是照搬原文：例如别人把 LLM 面试分成 Transformer、训练对齐、RAG、Agent、部署、手撕、项目，就对照本学习包看是否缺模块。
3. 提取“高频问题与追问”：只记录重复出现、容易答虚、能区分候选人深度的问题。
4. 对比本地章节，找薄弱点：哪里只有概念没有公式，哪里只有结论没有推导，哪里缺项目落地或面试表达。
5. 补充内容时统一写成本地原创讲解：通俗解释 + 深入机制 + 公式/流程 + 工程 trade-off + 高频问答 + 背诵版。
6. 保留来源链接和更新时间，方便之后回到原文核对更新。
7. 重建 `LLM学习.html`，并检查目录、搜索、Markdown 渲染和本地链接。

## 外部资料分类

### 1. 社交平台与面经

重点看这些信息：

- 2025、2026 大模型岗位、实习、项目实践和真实面经。
- 公司常见题型：算法岗、NLP 算法岗、LLM 应用岗、RAG/Agent 工程岗、推理部署岗的差异。
- 高频追问方式：面试官如何从一个概念继续追到公式、源码、复杂度、工程实现。
- 候选人踩坑：答得太虚、只会名词、公式不会推、项目指标说不清。

吸收方式：

- 不复制整段面经，只整理“题目、追问、考察点、推荐回答结构”。
- 对重复出现的问题优先补进 `03_高频题单100题.md` 和对应完整学习章。

### 2. GitHub 八股与学习仓库

重点看这些信息：

- 分类是否完整：基础 ML/DL/NLP、Transformer、LLM 训练、对齐、RAG、Agent、推理部署、手撕代码、项目。
- 题目覆盖是否贴近大模型岗位，而不是泛泛机器学习八股。
- 代码是否可运行，特别是 Attention、LoRA、RoPE、KV Cache、Beam Search、NMS、LRU、Top-k/Top-p 等。

吸收方式：

- 将优质仓库下载到 `外部资料_GitHub/` 作为原始资料。
- 本地学习章只保留原创讲解、题单索引和必要链接。

### 3. 官方文档与论文

重点看这些信息：

- 官方文档：Transformers、PyTorch、vLLM、llama.cpp、LangChain/LangGraph、向量数据库、MCP 等。
- 代表论文：Attention is All You Need、LoRA、DPO、PPO/RLHF、RoPE、FlashAttention、PagedAttention、RAG、GraphRAG、MoE、DeepSeek/GRPO/MLA 相关资料。

吸收方式：

- 用官方/论文核对概念、公式、复杂度和边界条件。
- 面试版讲解不堆论文细节，只保留“为什么提出、核心公式、解决什么问题、有什么代价、怎么落地”。

### 4. 课程、博客和专题教程

重点看这些信息：

- 哪些教程把复杂概念讲得清楚，例如 Self-Attention 到 Multi-Head Attention、KV Cache 推理流程、LoRA 参数注入、RAG chunking 和 rerank。
- 哪些图、例子、类比适合转成自己的理解。

吸收方式：

- 只吸收讲解思路和知识结构。
- 本地内容要重新组织成适合系统学习和阶段性冲刺的顺序。

## 每章持续补强方向

### 01 Transformer 与 Attention

继续补：

- token、embedding、position、Q/K/V、attention score、softmax、weighted sum、MHA、FFN、残差、LayerNorm、LM Head 的完整计算链。
- 每一步的矩阵 shape、公式、参数量、计算复杂度。
- RoPE、ALiBi、GQA/MQA/MLA、FlashAttention、KV Cache 的联系和区别。
- 高频追问：为什么除以 `sqrt(d_k)`、为什么要多头、为什么 causal mask、训练和推理 attention 的差异。

### 02 训练、微调与对齐

继续补：

- SFT、Instruction Tuning、LoRA/QLoRA、Prefix/Prompt Tuning 的区别。
- RLHF、PPO、DPO、ORPO、GRPO 的目标函数、训练流程和适用场景。
- 数据清洗、packing、loss mask、学习率、梯度累积、混合精度、显存估算。
- 高频追问：为什么 LoRA 有效、DPO 为什么不需要 reward model、过拟合怎么发现、微调后能力退化怎么办。

### 03 RAG、Agent 与 MCP

继续补：

- query rewrite、chunking、embedding、ANN 检索、rerank、context compression、answer synthesis 的完整链路。
- RAG 评估：召回率、命中率、faithfulness、answer relevance、人工评审、线上日志闭环。
- Agent 的 planning、tool calling、memory、reflection、multi-agent、失败恢复和权限控制。
- MCP 的定位、资源/工具/提示词模型、安全边界和项目落地话术。

### 04 推理部署与性能优化

继续补：

- Prefill/Decode、吞吐/延迟、batching、continuous batching、KV Cache、PagedAttention。
- Tensor Parallel、Pipeline Parallel、Data Parallel、Expert Parallel 的使用边界。
- 量化：INT8、INT4、GPTQ、AWQ、SmoothQuant、KV Cache 量化的 trade-off。
- 高频追问：为什么 decode 慢、为什么 vLLM 快、batch 变大会怎样、显存瓶颈怎么定位。

### 05 手撕代码

继续补：

- 每个高频手撕题给出最小可运行版本、复杂度、边界条件和面试讲解。
- PyTorch 手写 Self-Attention、MHA、LayerNorm、RMSNorm、LoRA Linear、RoPE、top-k/top-p sampling。
- 工程题：LRU、限流器、线程池、生产者消费者、日志解析、向量检索简化实现。

### 06 项目面经与冲刺

继续补：

- 按项目类型准备表达：RAG 问答、Agent 工具调用、模型微调、推理加速、多模态应用。
- 每个项目形成“背景、问题、方案、指标、难点、复盘、追问”七段式。
- 补公司/岗位差异：算法岗重公式和实验，工程岗重系统设计和稳定性，应用岗重业务指标和交付。

### 07 ML/DL/NLP 公共基础

继续补：

- 交叉熵、KL、softmax、梯度、优化器、正则化、归一化、初始化、评估指标。
- BERT/GPT、Seq2Seq、BPE/WordPiece/SentencePiece、N-gram、CRF 等老基础如何接到 LLM 面试。
- 高频追问：AUC 怎么算、Adam 为什么有效、BatchNorm/LayerNorm 区别、梯度消失爆炸怎么处理。

### 08 2025-2026 大模型热点

继续补：

- DeepSeek、MoE、MLA、GRPO、Reasoning Model、长上下文、Context Engineering、GraphRAG、多模态、端侧模型、安全与评测。
- 每个热点都按“是什么、为什么火、核心机制、面试怎么问、项目怎么用、常见误区”整理。
- 区分“论文热点”“产业热点”和“高频实战热点”，避免为了新而新。

## 内容质量验收

每次补充后，用下面清单自检：

- 是否能让阶段性学习者直接读懂，而不是只给关键词。

- 是否有面试口语版回答。
- 是否有追问和误区。
- 是否能连接到项目经历。
- 是否避免整段复制外部资料。
- 是否更新来源链接、更新日志和 `LLM学习.html`。
